A prototyping-first design system for editorial websites.

Built with modern CSS. Rooted in a philosophy where nothing gets thrown away.

The Problem

Web design has traditionally been wasteful. We spend hours in Figma crafting comps that get handed off and translated. We configure Tailwind only to fight against it later. We build wireframes in tools that don't reflect reality, then throw them away when the next phase begins.

How much better would our work be if we could turn those hours into real design thinking and execution?

The Solution

Live Wires takes a different approach: everything you build takes you closer to the final product. Start with real content in real HTML. The baseline styles make it look good immediately. Adjust tokens to establish your foundation. Layer utilities for art direction. Extract components as patterns emerge.

Nothing is throwaway. Each step builds on the last. The prototype becomes the product.

Instead of creating a series of disposable deliverables, the design process becomes an evolution.

Good Defaults + Additive Art Direction

This page demonstrates that semantic HTML looks good with minimal classes. The baseline typography, spacing, and layout just work. You don't fight the system to make things work. You add to it.

The three-tier approach:

  1. Semantic HTML—Headings, paragraphs, lists look good by default
  2. Layout primitives—.stack, .grid, .cluster for composition
  3. Utilities—.mt-4, .text-center, .scheme-warm for art direction

The Sacred Baseline

Everything in Live Wires derives from the --line variable. Change the base font size or line height, and the entire system recalculates. This ensures perfect vertical rhythm across your design.
