The principle

Edit tokens first, then elements. The cascade handles everything else.

Most theming happens in src/css/1_tokens/. For deeper typographic customization, you may also edit src/css/4_elements/. Never edit component CSS for theming.

Analyze the source

Before writing CSS, extract the visual DNA from your source design. Whether it's a website, PDF, or screenshot, look for:

Colors

  • Primary brand color. The dominant color for key UI elements.
  • Secondary colors. Supporting colors for accents, CTAs, backgrounds.
  • Background/foreground pairs. Light and dark combinations.
  • Specific hex values. Use browser dev tools, a color picker, or PDF inspector.

Typography

  • Font family. Check Google Fonts, Adobe Fonts, or inspect the source.
  • Weights in use. Light, regular, bold, etc.
  • Heading sizes. Visual hierarchy from h1 to body text.
  • Body text size. The base reading size.

Typographic conventions

Study how the source handles common elements:

  • Headings. Weight, case (normal/uppercase/small caps), tracking, decoration.
  • Blockquotes. Left border, background, quotation marks, font style.
  • Form fields. Border style, radius, padding, focus state.
  • Buttons. Shape (sharp/rounded/pill), text weight, padding, style (solid/outline).

Other details

  • Border radius. Sharp, subtle rounding, or pill shapes.
  • Spacing. Tight, comfortable, or generous whitespace.

Build your palette

Live Wires uses 11-step color scales from 50 (lightest) to 950 (darkest). Update the existing scales. Don't invent new color names.

Use existing variables

Update --color-green-*, --color-blue-*, etc. in color.css. If your brand is forest green, update the green scale. Don't create --color-forest-*.

The 500 anchor

Position your primary brand color at the 500 level. This gives you:

  • 50-400: Lighter shades for backgrounds, hover states, subtle accents
  • 600-950: Darker shades for text, borders, dark mode, depth

Brand positioning

Some brands naturally sit elsewhere in the scale:

  • Light brands (pastels, bright colors): Anchor at 300-400
  • Dark brands (forest greens, navy blues): Anchor at 600-700

This is fine. Just make sure you have headroom above and below.

Smooth interpolation

Avoid jarring jumps between adjacent values:

/* Bad: Jumps from light to dark */
--color-green-300: #88CEBA;  /* Light mint */
--color-green-400: #2A7A5E;  /* Suddenly dark */

/* Good: Smooth progression */
--color-green-300: #88CEBA;
--color-green-400: #5FB89E;
--color-green-500: #3D9A7E;
--color-green-600: #2A7A5E;

Update tokens

Semantic colors

Map your scale to semantic tokens in :root:

:root {
  --color-bg: var(--color-white);
  --color-fg: var(--color-green-700);
  --color-accent: var(--color-green-600);
  --color-muted: var(--color-green-500);
  --color-border: var(--color-green-700);
  --color-subtle: var(--color-green-100);
}

Typography

In typography-base.css, update font stacks:

--font-sans: 'Inter', -apple-system, BlinkMacSystemFont, sans-serif;
--font-serif: 'Merriweather', Georgia, serif;

--font-body: var(--font-sans);
--font-heading: var(--font-sans);

Font weight mapping

If your font has limited weights, map the weight variables accordingly:

/* Font only has 400 and 700 */
--font-light: 400;
--font-normal: 400;
--font-medium: 400;
--font-semibold: 700;
--font-bold: 700;

Loading web fonts

Google Fonts or Adobe Fonts: Add to public/_includes/head.html:

<link rel="preconnect" href="https://fonts.googleapis.com">
<link href="https://fonts.googleapis.com/css2?family=Inter:wght@400;500;700&display=swap" rel="stylesheet">

Self-hosted fonts: Add @import to typography-base.css:

@import url('/fonts/your-font.css');

Create schemes

Schemes override semantic colors for specific sections. Define them in color.css.

Naming convention

  • .scheme-{color} for primary variant
  • .scheme-{color}-light for light background
  • .scheme-{color}-dark for dark background

Light-on-dark schemes

.scheme-forest {
  --color-bg: var(--color-green-700);
  --color-fg: var(--color-white);
  --color-accent: var(--color-green-300);
  --color-muted: var(--color-green-200);
  --color-border: var(--color-green-300);
  --color-subtle: var(--color-green-800);
  --vf-grad: -50;
}

Dark-on-light schemes

.scheme-mint {
  --color-bg: var(--color-green-100);
  --color-fg: var(--color-green-700);
  --color-accent: var(--color-green-600);
  --color-muted: var(--color-green-500);
  --color-border: var(--color-green-500);
  --color-subtle: var(--color-green-200);
  --vf-grad: 0;
}

The vf-grad variable

Controls variable font grade for optical adjustment:

  • 0 for light backgrounds
  • -25 to -50 for dark backgrounds (reduces perceived heaviness)

Dark mode

When you properly update the schemes and :root semantic colors, dark mode works automatically via the prefers-color-scheme media query and .dark-mode class. No separate rules needed.

Heading hierarchy

Match the source heading sizes using existing text-* variables. Edit src/css/4_elements/typography.css.

h1 { font-size: var(--text-4xl); }
h2 { font-size: var(--text-2xl); }
h3 { font-size: var(--text-xl); }
h4 { font-size: var(--text-lg); }
h5 { font-size: var(--text-base); }
h6 { font-size: var(--text-sm); }

Available sizes

text-xs, text-sm, text-base, text-lg, text-xl, text-2xl, text-3xl, text-4xl, text-5xl, text-6xl, text-7xl, text-8xl, text-9xl

Only use these. Don't create new size variables.

Foundational elements

Match the source's typographic conventions by editing element styles. Use existing variables for sizing and spacing.

Headings beyond size

Headings carry typographic character beyond just size. In typography.css:

h1, h2, h3, h4, h5, h6 {
  font-weight: var(--font-semibold);
  text-transform: none;  /* Or uppercase */
  letter-spacing: var(--tracking-lg);
}

/* Add decoration if source uses it */
h2 {
  border-block-end: var(--line-1px) solid var(--color-border);
  padding-block-end: var(--line-05);
}

Blockquotes

Match the source's quotation treatment in quotes.css:

blockquote {
  /* Border style */
  border-inline-start: var(--line-025) solid var(--color-accent);

  /* Or use background instead */
  background: var(--color-subtle);
  padding: var(--line-1);

  /* Typography */
  font-style: italic;  /* Or normal */
}

Form fields

Match input styling in forms.css:

input[type="text"],
textarea,
select {
  border: 1px solid var(--color-border);
  border-radius: var(--radius-normal);
  padding: var(--line-05);

  &:focus-visible {
    outline: 2px solid var(--color-accent);
    border-color: var(--color-accent);
  }
}

Buttons

Match button shape and proportions in forms.css:

:root {
  --button-h-space: var(--line-1);
  --button-v-space: var(--line-05);
  --button-radius: var(--radius-loose);
  --button-font-size: var(--text-sm);
}

button, .button {
  font-weight: var(--font-semibold);
  text-transform: uppercase;  /* If source uses it */
  letter-spacing: 0.05em;
}

Available variables

Spacing
--line-025, --line-05, --line-075, --line-1, --line-15, --line-2
Radius
--radius-none, --radius-tight, --radius-normal, --radius-loose, --radius-full
Weights
--font-light, --font-normal, --font-medium, --font-semibold, --font-bold

Accessibility

Verify your color combinations meet WCAG contrast requirements.

Minimum ratios

Body text
4.5:1
Large text (18px+ or 14px+ bold)
3:1
UI components
3:1

Quick reference

Background Minimum foreground
100-200 (very light)700+ or black
300-400 (medium-light)800+ or black
500 (medium)white or 100
600-700 (medium-dark)white or 100-200
800-900 (dark)white or 100-300

Testing

  • Browser DevTools (Chrome and Firefox have built-in contrast checking)
  • WebAIM Contrast Checker
  • Accessible Colors

Common mistake

Muted text often fails contrast. Check --color-muted carefully on medium-toned backgrounds.

Clean up

After updating tokens, review and clean up the Manual documentation.

Pages to check

  • /manual/brand/colors.html to update color swatches
  • /manual/components/schemes.html to remove unused schemes and add new ones
  • /manual/brand/fonts.html to update font samples

Delete legacy schemes

If your theme uses greens, remove the orange, blue, and other unused schemes from:

  1. src/css/1_tokens/color.css
  2. The Manual scheme documentation

Keep the codebase clean. Don't ship unused CSS.
