Live Wires Guide

A prototyping-first design system for editorial websites. Built with modern CSS and minimal dependencies.

Philosophy: Good defaults + additive art direction. Semantic HTML looks good for prototyping out of the box, utility classes provide precise control for art direction.

How do I use this guide?

This guide documents Live Wires' CSS architecture, layout primitives, and utility classes. Use it as reference while prototyping. Each section links to the actual source files.

Start here: Write semantic HTML in public/index.html. It'll look good with zero classes. Add layout primitives (.stack, .grid) for composition. Use utilities for precise art direction.

Setup

Live Wires uses Vite for development with instant HMR (Hot Module Replacement). No build step during development, just instant updates.

What Vite does:

  • Instant CSS updates (no refresh needed)
  • Autoprefixer for vendor prefixes
  • Dev server with automatic browser sync
  • Optimized production builds

Getting started:

Step 1
Install Node.js (18+ required)
Step 2
Clone this repository and navigate to the project folder
Step 3
Run npm install to install dependencies
Step 4
Run npm run dev to start the development server
Step 5
Open http://localhost:3000 in your browser

Core commands:

  • npm run dev - Start dev server (run this when working)
  • npm run build - Build optimized CSS to dist/
  • npm run preview - Preview production build

Project structure

Each Live Wires installation is a single prototype. No templates, no build complexity. Just HTML, CSS, and a simple Web Component for includes.

Directory overview:

/src/css/
All CSS source files organized by cascade layers (tokens, elements, layouts, utilities)
/src/css/1_tokens/
Design tokens: spacing.css, typography.css, color.css
/src/css/4_elements/
Semantic HTML defaults (headings, paragraphs, links, lists, forms, tables)
/src/css/5_layouts/
Layout primitives: box, stack, grid, cluster, sidebar, center, and section
/src/css/7_utilities/
Utility classes: spacing, typography, color
/src/main.js
Vite entry point (loads CSS and Web Components)
/src/html-include.js
Zero-dependency Web Component for HTML includes
/public/
Your prototype lives here. Build HTML pages directly in this folder.
/public/_includes/
Reusable HTML fragments (header, footer, etc.) used with <html-include>. Customize and covert easily for your own front-end system.
/public/fonts/
Web font files
/public/img/
Images and graphics
/public/guide/
This style guide (you're reading it). Also functions as a component library.

Markup

Write semantic HTML5. That's it. Your content will look good with zero classes.

The three-tier approach:

  1. Semantic HTML - Headings, paragraphs, lists look good by default
  2. Layout primitives - Add section, box, .stack, .grid, .cluster for composition
  3. Utilities - Use .mt-4, .text-center, .scheme-warm for art direction

HTML Includes

Live Wires uses a zero-dependency Web Component for includes. Load the script first, then use <html-include> tags:

<!DOCTYPE html>
<html lang="en">
  <head>
    <!-- Load Web Component first -->
    <script type="module" src="/src/main.js"></script>

    <!-- Now includes work -->
    <html-include src="/_includes/head.html"></html-include>
  </head>
  <body>
    <html-include src="/_includes/header.html"></html-include>

    <main>
      <!-- Your content -->
    </main>

    <html-include src="/_includes/footer.html"></html-include>
  </body>
</html>

Note: Includes are fetched at runtime, not pre-rendered. Use for repeated UI (header, footer), keep unique content in main HTML.

CSS

Live Wires only native CSS. No Sass, no preprocessors. Just modern CSS features supported in all current browsers.

CSS Cascade Layers

Live Wires uses CSS Cascade Layers for explicit specificity control. This means utilities always win over components, components over layouts, etc. No !important needed.

@layer reset, base, layouts, components, utilities;

Layer hierarchy:

  1. reset - Browser normalization (reset.css)
  2. base - Semantic HTML defaults (4_elements/)
  3. layouts - Layout primitives (5_layouts/)
  4. components - UI components (6_components/)
  5. utilities - Single-purpose classes (7_utilities/)

Typography-first

All spacing and measurement units are derrived from the document's baseline: the --line custom property defined in spacing.css:

:root {
  --text-base: clamp(1rem, 0.9rem + 0.5vw, 1.125rem);
  --line-height-ratio: 1.5;
  --line: calc(var(--text-base) * var(--line-height-ratio));

  /* All spacing is multiples of --line */
  --space-0: 0;
  --space-1: calc(var(--line) * 0.5);
  --space-2: var(--line);
  --space-3: calc(var(--line) * 1.5);
  --space-4: calc(var(--line) * 2);
}

Change --text-base or --line-height-ratio and all spacing throughout the system recalculates automatically.

Modern CSS Features

  • Cascade Layers - Explicit specificity control
  • Native Nesting - No preprocessor needed
  • Container Queries - Components respond to their container
  • Logical Properties - margin-block-start instead of margin-top
  • Fluid Typography - clamp() for smooth scaling
  • Custom Properties - Variables throughout

Design Tokens

All design tokens are CSS custom properties in 1_tokens/:

spacing.css
Baseline (--line), spacing scale, layout tokens
typography.css
Type scale, font stacks, weights, reading measures
color.css
Base palette, semantic colors, color schemes, automatic dark mode

PostCSS Configuration

Minimal PostCSS in postcss.config.js. Just Autoprefixer, nothing else:

export default {
  plugins: {
    autoprefixer: {}
  }
};

That's it. No preprocessing, no transformations. Native CSS all the way.

Utility Classes

Tailwind-inspired utilities for precise control. All utilities use the utilities cascade layer, so they always win.

Spacing Utilities (spacing.css)

.mt-4
margin-block-start: var(--space-4)
.mb-2
margin-block-end: var(--space-2)
.px-3
padding-inline: var(--space-3)
.py-2
padding-block: var(--space-2)

Available scales: 0 through 6 (based on --line multiples)

Typography Utilities (typography.css)

.text-4
Font size: var(--text-4)
.font-bold
Font weight: 700
.measure
Max width: 65ch (optimal reading length)
.text-center
Text align: center

Color Schemes (color.css)

.scheme-warm
Applies warm color palette
.scheme-cool
Applies cool color palette
.scheme-dark
Applies dark color palette

Layout Utilities (grid.css)

.flex
display: flex
.grid
display: grid
.gap-3
gap: var(--space-3)

Javascript

Live Wires uses minimal, native JavaScript. No jQuery, no frameworks.

What's included

/src/main.js
Vite entry point - loads CSS and Web Components
/src/html-include.js
Zero-dependency Web Component for HTML includes (uses native fetch() API)

Adding third-party scripts

Use npm to install packages, then import in main.js:

npm install package-name
// In src/main.js
import 'package-name';

Vite handles the rest automatically.

Compatibility

Live Wires supports all modern browsers (last 2 versions). This includes:

  • Chrome/Edge (Chromium)
  • Firefox
  • Safari

Modern CSS features

Live Wires uses cutting-edge CSS that requires current browsers:

  • CSS Cascade Layers - Safari 15.4+, Chrome 99+, Firefox 97+
  • Native CSS Nesting - Safari 16.5+, Chrome 112+, Firefox 117+
  • Container Queries - Safari 16+, Chrome 105+, Firefox 110+
  • :has() selector - Safari 15.4+, Chrome 105+, Firefox 103+

Autoprefixer

Vite automatically runs Autoprefixer via PostCSS. Configure browser targets in package.json:

"browserslist": [
  "last 2 versions",
  "not dead",
  "not ie 11"
]
