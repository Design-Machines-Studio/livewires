Live Wires Guide

A prototyping-first design system for editorial websites. Built with modern CSS, rooted in a philosophy of progressive design.

Philosophy

Web design has traditionally been a wasteful process. We spend hours in Figma crafting comps that get handed off and translated. We configure Tailwind only to fight against it. We build wireframes that get thrown away.

Live Wires takes a different approach: everything you build takes you closer to the final product.

Instead of creating disposable deliverables, the design process becomes an evolution. Nothing is throwaway—each step builds on the last.

Start with real content in real HTML. The baseline styles make it look good immediately. Adjust tokens to establish your visual foundation. Layer utilities for art direction. Extract components as patterns emerge.

This is designing in the browser, but without the friction. Semantic HTML looks good by default. You don't fight the system to make things work. You add to it.

The three-tier approach

  1. Semantic HTML—Headings, paragraphs, lists look good by default
  2. Layout primitives—.stack, .grid, .cluster, .sidebar for composition
  3. Utilities—.mt-4, .text-center, .scheme-warm for art direction

See the Design process page for the full workflow.

How to use this guide

This guide documents Live Wires' CSS architecture, layout primitives, and utility classes. But it's also a design tool—when you change tokens, browse these pages to see how your changes affect the entire system.

Start here: Write semantic HTML in public/index.html. It'll look good with zero classes. Add layout primitives for composition. Use utilities for precise art direction.

HTML Includes

Live Wires uses a zero-dependency Web Component for includes. Load the script first, then use <html-include> tags:

<!DOCTYPE html>
<html lang="en">
  <head>
    <!-- Load Web Component first -->
    <script type="module" src="/src/js/main.js"></script>

    <!-- Now includes work -->
    <html-include src="/_includes/head.html"></html-include>
  </head>
  <body>
    <html-include src="/_includes/header.html"></html-include>

    <main>
      <!-- Your content -->
    </main>

    <html-include src="/_includes/footer.html"></html-include>
  </body>
</html>

Note: Includes are fetched at runtime, not pre-rendered. Use for repeated UI (header, footer), keep unique content in main HTML. See the HTML Includes page for details.

CSS

Live Wires uses only native CSS. No Sass, no preprocessors. Just modern CSS features supported in all current browsers.

CSS Cascade Layers

Live Wires uses CSS Cascade Layers for explicit specificity control. This means utilities always win over components, components over layouts, etc. No !important needed.

@layer reset, base, layouts, components, utilities;

Layer hierarchy

  1. reset — Browser normalization (reset.css)
  2. base — Semantic HTML defaults (4_elements/)
  3. layouts — Layout primitives (5_layouts/)
  4. components — UI components (6_components/)
  5. utilities — Single-purpose classes (7_utilities/)

Typography-first

All spacing and measurement units are derived from the document's baseline: the --line custom property defined in spacing.css:

:root {
  --text-base: clamp(1rem, 0.9rem + 0.5vw, 1.125rem);
  --line-height-ratio: 1.5;
  --line: calc(var(--text-base) * var(--line-height-ratio));

  /* All spacing is multiples of --line */
  --line-05: calc(var(--line) * 0.5);
  --line-1: var(--line);
  --line-15: calc(var(--line) * 1.5);
  --line-2: calc(var(--line) * 2);
}

Change --text-base or --line-height-ratio and all spacing throughout the system recalculates automatically.

Modern CSS Features

  • Cascade Layers — Explicit specificity control
  • Native Nesting — No preprocessor needed
  • Container Queries — Components respond to their container
  • Logical Properties — margin-block-start instead of margin-top
  • Fluid Typography — clamp() for smooth scaling
  • Custom Properties — Variables throughout

Design Tokens

All design tokens are CSS custom properties in 1_tokens/:

spacing.css
Baseline (--line), spacing scale
typography-base.css
Type scale, font stacks, weights
color.css
Palette, semantic colors, schemes

PostCSS Configuration

Minimal PostCSS in postcss.config.js. Just Autoprefixer, nothing else:

export default {
  plugins: {
    autoprefixer: {}
  }
};

That's it. No preprocessing, no transformations. Native CSS all the way.

Utility Classes

Tailwind-inspired utilities for precise control. All utilities use the utilities cascade layer, so they always win.

JavaScript

Live Wires uses minimal, native JavaScript. No jQuery, no frameworks.

What's included

/src/js/main.js
Vite entry point — loads CSS and Web Components
/src/js/html-include.js
Zero-dependency Web Component for HTML includes

Adding packages

Use npm to install, then import in main.js:

npm install package-name
import 'package-name';

Vite handles the rest.

Compatibility

Live Wires supports all modern browsers (last 2 versions): Chrome/Edge, Firefox, Safari.

Modern CSS requirements

  • CSS Cascade Layers — Safari 15.4+, Chrome 99+, Firefox 97+
  • Native CSS Nesting — Safari 16.5+, Chrome 112+, Firefox 117+
  • Container Queries — Safari 16+, Chrome 105+, Firefox 110+
  • :has() selector — Safari 15.4+, Chrome 105+, Firefox 103+

Autoprefixer

Vite runs Autoprefixer via PostCSS. Configure targets in package.json:

"browserslist": [
  "last 2 versions",
  "not dead",
  "not ie 11"
]
