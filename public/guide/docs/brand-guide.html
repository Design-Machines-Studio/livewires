Customizing the brand guide

How to adapt the brand guide section to your project.

Overview

The brand guide is a living document that connects your brand identity to the design system. As you update tokens (colors, typography, spacing), the brand guide automatically reflects those changes.

This tight coupling ensures your documentation always matches your implementation—no manual syncing required.

Brand guide pages

  • Logos — Brand marks, lockups, usage guidelines
  • Colors — Color palette using the <color-swatch> component
  • Fonts — Typography using the <type-sample> component
  • Graphics — Illustration style, photography, iconography
  • Voice & tone — Brand personality and writing principles
  • Formatting — Text conventions, punctuation, capitalization
  • Audience — Target personas and content strategy

The color-swatch component

The <color-swatch> Web Component displays color information with automatic format conversion and accessibility considerations.

Basic usage

The simplest usage only requires a token name. The component automatically resolves the color value from CSS:

<!-- Just provide the token — color is resolved from CSS -->
<color-swatch token="color-accent"></color-swatch>

You can also provide an explicit hex value if needed:

<color-swatch
  title="Brand Blue"
  token="color-accent"
  hex="#0008FF"
></color-swatch>

Available attributes

token
CSS custom property name without -- prefix (e.g., color-accent). The color value is automatically resolved from CSS if hex is not provided.
title
Display name for the color (defaults to token name)
hex
Explicit hex color value. If omitted, resolved from the CSS token. RGB, HSLA, and OKLCH are auto-calculated.
scheme
Optional scheme class name (e.g., scheme-accent)
rgb, hsla, oklch
Override auto-calculated values with custom formats
cmyk, pms
Print color values (not auto-calculated)

Automatic features

  • CSS resolution — Color values are resolved from CSS custom properties automatically
  • Format conversion — Hex values are automatically converted to RGB, HSLA, and OKLCH
  • Text color — Light or dark text is chosen based on WCAG luminance calculation
  • Visual preview — The color is displayed as the swatch background

Example with print values

<color-swatch
  title="Pantone Blue"
  token="color-brand"
  scheme="scheme-brand"
  cmyk="100, 97, 0, 0"
  pms="Blue 072 C"
></color-swatch>

The type-sample component

The <type-sample> Web Component displays typography samples for documenting your type system. It uses the same token convention as <color-swatch>—pass CSS variable names without the -- prefix.

Basic usage

Provide token names for typeface and weight. The component resolves them from CSS automatically:

<!-- Just provide tokens — values resolved from CSS -->
<type-sample
  title="Body Text"
  typeface="font-sans"
  weight="font-normal"
  text
></type-sample>

Available attributes

title
Display name for the type style
typeface
Font family token without -- prefix (e.g., font-sans, font-serif). Resolves to var(--font-sans).
weight
Font weight token without -- prefix (e.g., font-bold, font-normal). Resolves to var(--font-bold).
italic
Boolean attribute—add to apply italic style
display
Boolean attribute—show large display sample (AaBbCc1234)
headline
Boolean attribute—show headline-size character set
text
Boolean attribute—show body-size character set

Example for a serif headline

<type-sample
  title="Headline Serif"
  typeface="font-serif"
  weight="font-bold"
  display
  headline
></type-sample>

Example with italic

<type-sample
  title="Serif Italic"
  typeface="font-serif"
  weight="font-normal"
  italic
  text
></type-sample>

Updating brand guide content

Each brand guide page uses placeholder content designed to be replaced with your actual brand guidelines.

Content patterns

Placeholders follow consistent patterns:

  • [Bracketed text] indicates content to replace
  • Instructional text in text-muted explains what to add
  • Example structures show recommended organization

Recommended workflow

  1. Start with Logos and Colors—these are foundational
  2. Update Fonts after configuring your typography tokens
  3. Write Voice & tone before other content pages
  4. Complete Audience and Formatting as you develop content strategy
  5. Add Graphics guidelines as your visual style matures

Connecting to CSS tokens

The brand guide should reference your actual CSS tokens. When you update a token value, the brand guide reflects that change automatically. Both components use the same convention—pass the token name without the -- prefix.

<!-- Colors — just provide the token name -->
<color-swatch token="color-accent"></color-swatch>

<!-- With a custom title -->
<color-swatch
  title="Accent"
  token="color-accent"
></color-swatch>

<!-- Fonts — same token convention -->
<type-sample
  typeface="font-sans"
  weight="font-normal"
  text
></type-sample>

<!-- Serif bold with display sample -->
<type-sample
  title="Display Serif"
  typeface="font-serif"
  weight="font-bold"
  display
></type-sample>

A living document

The brand guide is designed to evolve with your project. Unlike static PDF brand guides, this documentation lives alongside your code and updates automatically.

Benefits of code-based brand guides

  • Always current — Token changes propagate to documentation automatically
  • Version controlled — Track brand evolution through git history
  • Collaborative — Designers and developers work from the same source
  • Accessible — No special software needed to view or edit

Maintenance tips

  • Update documentation when you add new tokens or components
  • Keep code examples current with actual implementation
  • Review content pages quarterly for accuracy
  • Delete placeholder content you don't plan to use
